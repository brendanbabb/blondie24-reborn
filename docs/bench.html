Main-thread bench

Runs the same self-play tournament the Worker runs (POP_SIZE=6, 3 games per individual per gen, depth-4 minimax, paper scoring) — but synchronously on the page's main thread, not in a Worker. The button locks up the tab for a few seconds while it runs.

Goal: compare Chrome V8 main-thread throughput vs. the Worker (~480 ms/gen on your box) vs. Node (~200 ms/gen on the same JS). If main-thread ≈ Node, the Worker context is specifically the bottleneck. If main-thread ≈ Worker, Chrome V8 is just slower than Node V8 on this machine.
